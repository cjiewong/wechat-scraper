# 项目重构总结（2026-03-04）

## 重构目标

1. 参考 `we-mp-rss` 的做法，改为微信公众平台 API 抓取链路
2. 重做前端 UI
3. 对 `wechat-scraper` 做结构级重构

## 已完成内容

### 1) 抓取链路 API 化

- 移除旧的“搜狗页面选择器抓取”主流程
- 新增公众号平台接口调用：
  - `searchbiz` 搜索公众号
  - `appmsgpublish` 获取历史文章列表
- 保留文章详情页抓取用于正文提取与导出

### 2) 主进程与会话管理重构

- 主进程 IPC 全量重构，按“会话 -> 搜索 -> 拉取 -> 导出”拆分
- 新增本地会话持久化（`data/session.json`）
- 增加会话有效性检测与状态回传

### 3) 导出管线重构

- 统一导出流程：抓取正文 -> 图片处理 -> Markdown/PDF 输出
- COS 配置改为可选，缺失时自动降级为原图链接
- 文件命名与日期解析逻辑增强，减少异常日期导致的错误

### 4) UI 重设计

- 三栏布局：会话面板 / 公众号与文章面板 / 导出与日志面板
- 新视觉主题：渐变背景、信息层级、状态徽章、进度与统计卡片
- 交互重排：默认全选文章、一键导出、实时状态日志

### 5) 文档更新

- `README.md`、`使用指南.md`、`快速参考.md` 全部改为 API 版流程
