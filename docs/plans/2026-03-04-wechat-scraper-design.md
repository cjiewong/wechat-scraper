# WeChat Scraper API 重构设计（2026-03-04）

## 目标

- 从“页面选择器抓取（搜狗/手动浏览）”切换到“微信公众平台 API 拉取”
- 重构导出链路和会话管理，降低页面结构变更风险
- 重新设计 UI，减少操作路径并突出状态反馈

## 架构决策

### 1. 数据源

- 公众号搜索：`/cgi-bin/searchbiz`
- 文章列表：`/cgi-bin/appmsgpublish`
- 正文提取：文章链接 HTML（`mp.weixin.qq.com/s/...`）

原因：与 `we-mp-rss` 保持一致，减少对前端页面结构的强依赖。

### 2. 会话模型

- 输入并持久化 `token + Cookie + User-Agent`
- 启动后自动加载本地会话
- 提供会话检测能力（主页可达且非登录页）

### 3. 导出流水线

- 拉取列表 -> 抓取正文 -> 图片处理 -> 格式转换 -> 文件落盘
- COS 上传可选，未配置自动降级保留原图链接

### 4. 前端信息架构

- 左栏：会话配置
- 中栏：公众号搜索、分页过滤、文章选择
- 右栏：导出配置、进度、统计、日志

## 风险与处理

- 风控/频控：保留延迟、捕获 ret=200013
- 会话过期：统一错误提示并要求刷新会话
- 图片上传失败：不中断主流程，只记录失败映射

## 验证策略

- `node --check` 覆盖主进程、抓取模块、渲染逻辑
- `npm start` 启动级检查
- 运行时通过状态栏和日志观察接口响应与导出进度
